# CommandInterpreter.applyCommands

## Declaration

```typescript
public applyCommands(cmds: TableCommand[]): void
```

## Purpose

Use `applyCommands()` to execute multiple commands synchronously in batch. This is the preferred method for executing multiple commands as it's more efficient than calling `applyCommand()` individually.

**When to use:**
- When executing multiple table operations in sequence
- When applying commands generated by `TableCommandPlanner`
- When replaying saved command sequences
- When applying undo/redo stacks

**When NOT to use:**
- Do NOT use for async operations—use `applyCommandsAsync()` instead
- Do NOT pass empty arrays (though harmless, it's unnecessary)

## Parameters

### `cmds: TableCommand[]` (required)

Array of commands to execute in order.

**Valid values:**
- Non-empty array of `TableCommand` objects
- Can contain mix of `StructuralCommand` and `CellCommand`
- Commands are executed in array order

**Common mistakes:**
- Passing commands that depend on each other without considering order
- Not validating commands before execution

## Return Value

Returns `void` (no return value).

**Side effects:**
- Executes all commands in order
- Mutates the underlying state (specifics depend on implementation)
- No return value or indication of success/failure
- Failed commands may leave state partially modified

## Best Practices

1. **Validate commands before execution**
   - Check command validity before passing to `applyCommands()`
   - The method may not perform extensive validation
   - Invalid commands may cause undefined behavior

2. **Use batch over individual calls**
   - Always prefer `applyCommands()` over loop of `applyCommand()`
   - Reduces dispatch overhead
   - May enable optimizations in the interpreter

3. **Handle errors gracefully**
   - Consider wrapping in try-catch
   - Partial execution may leave inconsistent state
   - Implement rollback if needed

4. **Combine with TableCommandPlanner**
   - Use `TableCommandPlanner` to generate command sequences
   - Pass generated commands directly to `applyCommands()`

## Common Pitfalls

1. **Using loops instead of batch**
   ```typescript
   // BAD: Individual calls
   commands.forEach(cmd => interpreter.applyCommand(cmd))

   // GOOD: Batch execution
   interpreter.applyCommands(commands)
   ```

2. **Not handling partial failures**
   ```typescript
   // BAD: Assumes all commands succeed
   interpreter.applyCommands(commands)
   // State may be inconsistent if a command failed

   // GOOD: Handle errors
   try {
     interpreter.applyCommands(commands)
   } catch (error) {
     // Handle or rollback
     console.error('Command execution failed:', error)
   }
   ```

3. **Empty command arrays**
   ```typescript
   // BAD: Unnecessary method call
   interpreter.applyCommands([])

   // GOOD: Skip execution
   if (commands.length > 0) {
     interpreter.applyCommands(commands)
   }
   ```

## Complete Example

```typescript
import { BuildinStateInterpreter, TableState, TableCommandPlanner, TableCommand } from '@aptx/table-commands-generator'

// Setup
const table = new TableState(10, 10)
const interpreter = new BuildinStateInterpreter(table)
const planner = new TableCommandPlanner(table)

// Scenario 1: Basic batch execution
const commands: TableCommand[] = [
  { type: 'INSERT_ROW', index: 5, count: 2 },
  { type: 'SET_CELL_ATTR', row: 5, col: 0, attr: 'rowSpan', value: 2 },
  { type: 'SET_CELL_ATTR', row: 5, col: 0, attr: 'colSpan', value: 2 },
]

interpreter.applyCommands(commands)
console.log('All commands executed')

// Scenario 2: Commands from TableCommandPlanner
const mergeCommands = planner.merge(0, 0, 1, 2)
if (mergeCommands) {
  interpreter.applyCommands(mergeCommands)
  console.log(`Executed ${mergeCommands.length} commands for merge`)
}

// Scenario 3: Error handling with validation
function validateCommands(cmds: TableCommand[]): { valid: boolean; errors: string[] } {
  const errors: string[] = []

  cmds.forEach((cmd, index) => {
    switch (cmd.type) {
      case 'INSERT_ROW':
      case 'DELETE_ROW':
        if (cmd.index < 0 || cmd.index >= table.getRowCount()) {
          errors.push(`Command ${index}: Row index ${cmd.index} out of bounds`)
        }
        break
      case 'INSERT_COL':
      case 'DELETE_COL':
        if (cmd.index < 0 || cmd.index >= table.getColCount()) {
          errors.push(`Command ${index}: Column index ${cmd.index} out of bounds`)
        }
        break
      case 'SET_CELL_ATTR':
      case 'CLEAR_CELL_ATTR':
        if (cmd.row < 0 || cmd.row >= table.getRowCount() || cmd.col < 0 || cmd.col >= table.getColCount()) {
          errors.push(`Command ${index}: Cell (${cmd.row}, ${cmd.col}) out of bounds`)
        }
        break
    }
  })

  return {
    valid: errors.length === 0,
    errors,
  }
}

// Usage
const testCommands: TableCommand[] = [
  { type: 'INSERT_ROW', index: 5, count: 1 },
  { type: 'INSERT_ROW', index: -1, count: 1 }, // Invalid!
]

const validation = validateCommands(testCommands)
if (!validation.valid) {
  console.error('Invalid commands:', validation.errors)
} else {
  interpreter.applyCommands(testCommands)
}
```
